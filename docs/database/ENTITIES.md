# Entity Definitions

TypeORM entity schemas and their mappings. Source: `libs/server/data-access-todo/src/lib/database/schemas/`.

---

## User

Table: `user`  
Schema: `user.entity-schema.ts`  
Domain: `IUser` (`@full-stack-todo/shared/domain`)

### Columns

| Column    | Type     | Nullable | Notes                                      |
|-----------|----------|----------|--------------------------------------------|
| `id`      | UUID     | No       | Primary key; generated by TypeORM          |
| `email`   | String   | No       | Unique; used as login identifier           |
| `password`| String   | No       | Stored hashed (never plain text)           |

### Relations

| Relation | Type         | Target | Notes                          |
|----------|--------------|--------|--------------------------------|
| `todos`  | One-to-many  | `todo` | Cascade delete; inverse: `user` |

One user has many todos. Deleting a user removes all their todos.

---

## Todo

Table: `todo`  
Schema: `to-do.entity-schema.ts`  
Domain: `ITodo` (`@full-stack-todo/shared/domain`)

### Columns

| Column       | Type    | Nullable | Notes                                           |
|--------------|---------|----------|-------------------------------------------------|
| `id`         | UUID    | No       | Primary key; generated by TypeORM               |
| `title`      | String  | No       | Required; unique per user (see UNIQUE_TITLE_USER)|
| `user_id`    | UUID    | Yes      | Foreign key to `user.id`; join column for `user`|
| `description`| String  | Yes      | Optional details                                |
| `completed`  | Boolean | No       | Default `false`; whether the todo is done        |

### Relations

| Relation | Type        | Target | Notes                          |
|----------|-------------|--------|--------------------------------|
| `user`   | Many-to-one | `user` | Join column: `user_id`; inverse: `todos` |

Many todos belong to one user. Unique constraint `UNIQUE_TITLE_USER` on `(title, user_id)` prevents duplicate titles per user.

---

## Cascade behaviors

Cascades are defined on relations so that certain actions on one entity automatically apply to related entities.

| Parent    | Relation | Effect |
|-----------|----------|--------|
| **User**  | `todos`  | When a user is **deleted**, all their todos are deleted. |

- **Where it is defined**: `user.entity-schema.ts` → `relations.todos` → `cascade: true`.
- **Direction**: One-way from User to Todo. Deleting a todo does not affect the user.
- **Rationale**: A user owns their todos; removing the account should remove their data (no orphaned todos).
